# React TypeScript Guidelines

- Use functional components with hooks (useState, useEffect, useContext, etc.)
- Define prop types using TypeScript interfaces or type aliases
- Keep components focused and reusable
- Use consistent naming conventions (PascalCase for components, camelCase for functions)
- Memoize expensive computations with useMemo and useCallback
- Handle loading, error, and empty states explicitly
- Prefer composition over inheritance
- Keep component files under 300 lines when possible
